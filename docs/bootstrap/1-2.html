グリッドシステム

公式ドキュメントを見てみるとこんなことが書いてある。

グリッドシステムは、12 カラムのシステムと 5 段階のレスポンシブ、Sass と mixin、いくつかの定義されたクラスですべての図形とサイズのレイアウトを作成可能です。

グリッドシステムを利用するとレイアウトを思いのままに指定することができる...っぽい。それぞれの言葉について説明していこう。

５段階のレスポンシブとは

xs
<576px		 sm
≥576px		 md
≥768px		 lg
≥992px		 xl
≥1200px		 xxl
≥1400px
Container
max-width		 None(auto) 540px 720px 960px 1140px 1320px
class prefix .col- .col-sm- .col-md- .col-lg- .col-xl- .col-xxl-
カラム数 12
Gutter width 1.5rem(.75rem on reft and right)

上記のようなルールでブレークポイントを指定できるというものだ。

bootstrapでは小さいものから優先して指定される。
col-2のような指定の仕方は一番小さいブレークアウトの段階2列分という指定の仕方であり、
より大きなブレークアウトにおいて何も指定していないため、他の全ての状態でも初めの指定が適用されている。

コンテナの時のように開いている画面の幅を変えてみて欲しい

コンテナの幅が画面幅に応じて変化するように、colの幅が変化するだろう。

col-md-4はmd幅の条件に適していない小さな幅の時は何も適用されない。 また、col-mdやcol-lgのようにカラム数を特に指定せずにおくと、指定の条件下において自動で調整してくれる。ただし12カラムを超える場合はやはり改行されてはみ出す。
